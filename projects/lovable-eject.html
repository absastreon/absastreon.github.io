Lovable-Eject

Out of lock-in. A one-command escape hatch for Lovable projects.

The work

Lovable is a lovely way to prototype. But the moment a project grows past a weekend hack — real users, a custom domain, a need for granular control — the hosting lock-in starts to chafe. Exporting the code gives you files, but not a cleanly deployable project. You end up hand-editing configs for an afternoon.

Lovable-Eject is the escape hatch. One command takes a Lovable project and produces a clean, dependency-aware export ready to deploy to Vercel, Netlify, a VPS, or anywhere that runs Node. No leftover Lovable-specific files, no broken imports, no surprises. Free, open source, does one thing well.

Built because I needed it, released because other people probably do too.

npx lovable-eject One command

What it does

F.01

Clean export

Strips Lovable-specific config, normalises project structure, and outputs a standard repository you'd expect to find anywhere.

F.02

Host-agnostic output

Works out of the box on Vercel, Netlify, Cloudflare Pages, Railway, or a plain Node server. No vendor coupling.

F.03

Env-var preservation

Keeps your environment configuration intact, with a .env.example generated for collaborators.

F.04

Zero install

Runs via npx. No global install, no config file, no lock-in to another tool. Use it once and forget it.
