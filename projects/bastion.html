Bastion

Security for the AI-era builder — local, fast, free to start.

The work

Since the rise of AI-assisted code, even experienced engineers routinely ship hardcoded secrets, missing security headers, and unsafe default CORS policies. Stanford's research suggests AI-generated code contains roughly 40% more vulnerabilities than hand-written equivalents — and enterprise security tooling starts at £300/month, putting basic hygiene out of reach for indie developers and small teams.

Bastion is the tool I wished existed: a privacy-first scanner that runs on your machine via npx bastion-scan, checks for the twelve most common security gaps in seconds, and generates AI-ready fix prompts tailored to your exact stack. Paste them into Claude or ChatGPT and ship fixes the same day.

Open source under MIT. No accounts, no uploads, no cloud. Self-scans at 100/100, ships with 759 tests, and is distributed with npm provenance. The Pro tier adds extra checks and a dashboard, but the core scanner will always be free.

npx bastion-scan scan Try it now

What it checks

C.01

Configuration & secrets

.gitignore patterns, .env.example validation, hardcoded API keys (OpenAI, AWS, Stripe, etc.), and sensitive file detection before they hit your repo.

C.02

Security headers & transport

CSP, HSTS, X-Frame-Options, X-Content-Type-Options, SSL cert expiry, TLS version, HTTPS redirects. The stuff browsers expect.

C.03

CORS, rate limiting & auth

Catches wildcard CORS with credentials, missing rate-limit middleware, and unprotected API routes without any auth provider detected.

C.04

Code patterns & dependencies

SQL string concatenation, eval() usage, and a full npm audit pass with CVE references and upgrade paths.

C.05

AI-ready fix prompts

Every failing check generates a prompt tailored to your stack. Paste into your LLM of choice, get contextual fixes, ship.
